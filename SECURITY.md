## Сообщение об уязвимостях

Если вы обнаружили уязвимость в безопасности, пожалуйста, **НЕ** создавайте публичный issue.

Вместо этого:
1. Отправьте email на security@socialauto.com
2. Опишите уязвимость детально
3. Приложите шаги для воспроизведения
4. Укажите потенциальное влияние

Мы ответим в течение 48 часов и будем работать над исправлением.

## Лучшие практики безопасности

### Для разработчиков

1. **Никогда не коммитьте**:
   - Пароли
   - API ключи
   - Токены доступа
   - Приватные ключи
   - .env файлы

2. **Используйте**:
   - Environment variables для секретов
   - Сильные пароли (min 12 символов)
   - 2FA где возможно
   - HTTPS everywhere
   - Prepared statements для БД запросов

3. **Регулярно обновляйте**:
   - Зависимости (npm audit)
   - Docker images
   - OS пакеты

### Для пользователей

1. **Защита аккаунтов**:
   - Используйте уникальные пароли
   - Включите 2FA
   - Регулярно меняйте пароли
   - Используйте прокси для автоматизации

2. **Безопасное использование**:
   - Не делитесь credentials
   - Проверяйте SSL сертификаты
   - Логаутьтесь после использования
   - Следите за подозрительной активностью

## Известные риски

1. **Автоматизация соцсетей** - может привести к бану аккаунтов
2. **Rate limiting** - превышение лимитов может заблокировать IP
3. **Хранение credentials** - риск компрометации данных

## Рекомендации

- Используйте отдельные прокси для каждого аккаунта
- Настройте warm-up для новых аккаунтов
- Не превышайте рекомендуемые лимиты действий
- Регулярно проверяйте статус аккаунтов
- Делайте бэкапы данных

## Compliance

- Не использовать для спама
- Не нарушать права других пользователей
- Использовать ответственно и этично
